## Commit — 2026-06-06 03:16

fix(nvim): patch treesitter predicates, add smart hover, optimize LSP

Fixed critical nvim-treesitter predicate bug that broke syntax highlighting when captures resolved to node-lists instead of single nodes (Nvim 0.10+ query-match shape change). The archived nvim-treesitter repo never migrated set-lang-from-info-string! and downcase! directives, causing "attempt to call method 'range' (a nil value)" errors that poisoned entire parse trees and broke highlighting across buffers.

Re-registered both directives with node-list defensiveness (extract first node when capture is a table without .range method), enabling markdown fenced code block syntax highlighting to work reliably again.

Added smart_hover LSP function that aggregates responses from all attached clients, prioritizes by client name (tailwindcss > vtsls > ts_ls > eslint), and renders a single popup with proper treesitter syntax highlighting. Unwraps single-fence markdown bodies (common vtsls shape) to display clean TypeScript/CSS without visible fence delimiters. Manually applies conceallevel and filetype since open_floating_preview's stylization path requires vim.g.syntax_on (not set in treesitter-only configs).

Replaced prettierd+eslint_d formatter chain with oxfmt (30x faster Rust formatter) for JS/TS/JSON. Disabled format-on-save to prevent cursor jumps and performance issues in large files. ESLint kept for JSDoc enforcement and advanced linting, but auto-fix disabled. Removed remark_ls (exit code 1 errors), keeping mdx_analyzer for both MDX and markdown.

Added LSP performance optimizations: request timeouts (5s for definitions, 10s for references in monorepos), disabled document highlighting in large files, colon-method syntax for Nvim 0.12+ client:request_sync. Created lsp-performance.lua, memory-monitor.lua, and lsp-debug.lua utilities for monitoring.

Deleted legacy agent-os-base/ and dot_agent-os/ directories (replaced by pi-agent-base structure). Updated pi-agent-base with new skills (design, jsdoc-conventions, spec-structure, sync-project-harness, sync-tmux-layout), expanded 0-project-init to include design assets phase, and enhanced sync-harness to track design folder changes.

Updated tmux.conf to bind `c` to copy-mode instead of new-window. Switched Pi default model to fireworks/kimi-k2p6. Enhanced run_after_sync-pi-agent-base.sh to render .tmpl files through chezmoi execute-template with Bitwarden integration.
