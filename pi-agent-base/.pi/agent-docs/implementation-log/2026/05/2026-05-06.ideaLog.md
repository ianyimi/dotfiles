- This harness is scoped narrowly to: editing the global Pi agent config (extensions, prompts, skills, templates) that gets deployed to `~/.pi/agent/` via chezmoi

## Stack
- TypeScript (top-level extensions loaded by Pi at runtime via esbuild — no build step)
- TypeScript sub-package: `extensions/_vim-powerline/` (has its own `package.json`, `tsconfig.json`, vitest)
- Markdown for prompts, skills, templates
- chezmoi sync via `run_after_sync-pi-agent-base.sh.tmpl` (in parent dir)

## Key decisions

1. **Workflow tier: low-care** — Agent implements changes directly. Exception list: changes to global `AGENTS.md`, `settings.json` `packages[]` order, and `prompts/0-project-init.md` must be flagged for explicit user review since they affect every Pi session globally.

2. **Verification loop: typecheck + (sub-pkg) build/test → user runs `cma` → user restarts Pi.** Agent does not run `cma` itself (requires Bitwarden session). Documented in `.pi/agent-docs/standards/verification.md`.

3. **Project-local override of `/learn` and `/research`** — both prompts now consult Pi's installed docs at `/Users/zaye/.local/share/fnm/.../pi-coding-agent/docs/` and `examples/` first, before any external sources. Topic-to-doc map captured in `.pi/agent-docs/standards/pi-apis.md`.

4. **Updated `run_after_sync-pi-agent-base.sh.tmpl`** to add `--exclude='.pi/'` so this meta-harness is not deployed as the global harness on `cma`. Without this exclusion, `~/.pi/agent/` would gain a `.pi/` subdir shadowing the real global config.

5. **Commit scope: `pi-base`** — because `pi-agent-base/` is a sub-tree of the chezmoi dotfiles repo, commits go to the parent repo. The `/commit` prompt is configured to `cd` up and scope commits with `type(pi-base): ...`.
