## Commit — 2026-04-22 20:39

feat(pi): add browser-annotations and pi-smart-fetch, fix cma bw-session chain

Add `@browser-annotations/pi` and `pi-smart-fetch` to the global pi package
list and mirror the installs in the macOS ansible playbook so `apConfig`
provisions them end-to-end. The browser-annotations pi package is only the
receiver half of the system, so the playbook also runs `npx --yes
browser-annotations@latest` to install the companion Chrome DevTools
extension that exposes the Feedback tab — without it, the pi-side server
has nothing to talk to.

Document `dot_zshrc.tmpl` as a key file in `.pi/AGENTS.md` with a table of
frequently-used aliases (`cma`, `apConfig`, `cm`, `nrd`, etc.) and the
`cd<Name>` / `nv<Name>` / `lg<Name>` / `aeromon<Action>` navigation
patterns, plus a rule telling agents to treat alias shorthands literally
rather than expanding them. Zaye routinely refers to commands by alias, so
agents need to grep this file first when they see an unfamiliar short
command.

Fix a latent bug in the `cma` chain: `bw-session-check`'s early-exit
"session valid" branch never wrote `~/.bw-session`, so if the bridge file
ever got deleted the subsequent `source ~/.bw-session` in the alias would
exit 1 and short-circuit the `&&` chain — `chezmoi apply` silently never
ran. The script now self-heals by recreating the file when the session is
valid but the file is missing, and the `cma` alias wraps the source step
in `{ ... || true; }` as belt-and-suspenders so a transient missing file
can't break the chain again. This is critical because `chezmoi apply`
relies on `BW_SESSION` in the parent shell to resolve `{{ bitwarden ... }}`
template functions.

Also reconcile drift in `pi-agent-base/settings.json`: bump
`defaultModel` to `claude-opus-4-7`, `lastChangelogVersion` to `0.68.1`,
and add the `suppressWarnings` block so the next rsync-based sync doesn't
clobber the live config.
